Install

System requirements?

Supports Linux (x64 / arm64 / armv7 / riscv64), macOS (Intel / Apple Silicon), Windows, WSL2, Alpine. Prebuilt binaries preferred; source compilation requires Rust ≥ 1.75(the installer handles Rust automatically).

Do I need root / admin?

install.sh currently requires root(因为要装编译工具链). Windows install.ps1 needs admin for Build Tools.npm method(npm i -g)doesn't need root, but the system-level command still requires privileges at the angles install step.

Difference between the four install methods? Same binary?

Identical — all produce ~/.local/bin/angles Rust binary:

  • npm: 打包了Install器, 跨平台, 国内走 npmmirror 镜像最快
  • curl / wget: pulls install.sh directly, best for Linux/macOS/WSL2 servers
  • PowerShell irm: native Windows first choice
Install interrupted halfway?

重Run同一条Install命令即可, Install器是幂等的: 已装的依赖会跳过, 不足的会补齐. config文件 ~/.angles/config.json is never overwritten.

GitHub raw blocked / slow in China?

Use npm method最稳(Default走 npmmirror). 或给_install.sh_ 换 GitHub 加速前缀, 例如: 

curl -fsSL https://ghfast.top/https://raw.githubusercontent.com/ZSJ305/angles-cli/main/install.sh | bash

Or set a proxy in terminal:export https_proxy=http://127.0.0.1:7890

config

Where is the config file?

~/.angles/config.json(Windows: %USERPROFILE%\.angles\config.json). Delete it to reset — next launch triggers the Gateway wizard.

Is my API key uploaded anywhere?

No.Keys are stored locally in config.json, or via ANGLES_API_KEY env var.所有请求直接发往your configured provider, Angles 不经过任何中转或代理服务器.

怎么重新config(换模型 / 换语言)？

Run angles gateway to re-enter the 5-step TUI wizard, change any field.Or edit config.json directly.view current config: angles config.

Approval policies: untrusted / on-request / never?

untrusted(default): auto-execute safe commands, ask before destructive ops.on-request: agent decides when to ask.never: auto-execute all — fastest but riskiest.angles-deletefile, rm -rf, git push --force always require confirmation regardless of policy.

Models & Providers

Which providers are supported?

OpenAI, Claude (Anthropic), Gemini (Google), DeepSeek, Grok (xAI), MiniMax, OpenRouter, 通义千问 Qwen, 智谱 GLM, Kimi (Moonshot), 自defs. The first 10 have preset base_url and default models; Custom lets you specify any endpoint.

Can I use local Ollama / vLLM / LM Studio?

Yes. These local inference services provide OpenAI-compatible endpoints. Select "Custom" in Gateway:

  • Base URL: http://localhost:11434/v1(Ollama)
  • API Key: anything (usually not validated locally)
  • Protocol: chat
  • Model: your model name, e.g. llama3.1
Claude and Gemini use different protocols — how?

Angles 在 api.rs adapts three protocols: chat(OpenAI-compatible), anthropic(Messages API), gemini(原生 generateContent). selecting Claude / Gemini auto-sets wire_api, no manual config needed.

How to switch models without changing other settings?

Edit config.json — change the model field to the new model ID. Other fields stay.或重新Run angles gateway and only change Step 3.

Tools & Usage

What are the 30+ tools?

6 groups: 文件创建(4), 读取搜索(5), 修改删除(7), 目录项目(5), 终端执行(3), 网络搜索(2), Git(5). Full reference at Tools page, searchable and filterable by category.

Why prefer replace over writefile?

angles-replace is a precise diff — clear intent, minimal change;writefile overwrites the entire file, risking unrelated content.The system prompt explicitly requires the agent to prefer replace > writefile.

How to make the agent run shell commands?

The agent autonomously calls angles-run, 你可以直接让它"Run测试""起服务看一下日志"之类. Whether confirmation is needed depends on your approval policy.Background tasks use angles-runbg, (returns PID);angles-kill terminates.

Can it search the web?

Yes.Gateway 第 5 步select a search engine:Bing / Baidu / Google / Yahoo / 自defs URL / 关闭. The agent calls angles-websearch, Bing is usually most stable.

Does it conflict with Codex CLI / Claude Code?

No.Angles is an independent binary angles, config and sessions are in ~/.angles/, no interference with other tools.You can run multiple agentic CLIs in different terminals.

Safety & Approval

Which operations always require confirmation?

Regardless of approval policy, these always require confirmation:

  • angles-deletefile — permanent file deletion
  • angles-run 配 rm -rf / sudo / mkfs / dd or similar destructive commands
  • angles-run git push --force
  • Any operation modifying files outside the workspace
Can the agent delete files recklessly?

Default untrusted policy: no — all writes require confirmation.Even in never, deletefile 和 rm -rf still force confirmation.系统提示also requires the agent not fix unrelated bugs or restructure files.

它会在我的项目目录外乱Run吗？

Default作Use域是当前工作目录及其子树, 修改工作区外文件需要确认. AGENTS.md specs also make the agent respect repo-specific constraints.

Troubleshooting

No response / network error?

先Run angles doctor first:

  • Check config.json completeness
  • Test connectivity to base_url
  • Verify API Key validity
  • Check if daily token budget is exhausted

Common causes: expired API Key, wrong base_url, missing proxy, daily budget exhausted (resets next day).

Getting grep: unrecognized option -P?

你大概率在 Alpine / BusyBox 上Run install.sh 的检测代码. BusyBox grep does not support PCRE(-P). This is a known issue in install.sh 内部对 Rust 版本号解析的已知兼容问题, 不影响实际功能 — Rust 版本检查会退化但does not block installation.

PowerShell execution policy error on Windows?

Use irm ... | iex 形式(管道执行)可绕过执行策略; 或在管理员 PowerShell 里Run Set-ExecutionPolicy -Scope CurrentUser RemoteSigned.

Can't download prebuilt binary?

The installer automatically falls back to source compilation: install Rust + cargo build --release. Takes a few minutes but guarantees success.You can also ANGLES_REPO=... git clone and build locally cargo build --release then copy manually.
